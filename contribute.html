How to get involved

Defold is developed by a team of developers from the Defold Foundation. As a free to use, source available, project we also accept community contributions to any part of the product, including, but not limited to, the engine, editor, extensions and documentation.

The source code is available on GitHub and it is divided into multiple repositories. Below is a list of the main repositories:

  • Engine and editor
  • Extension build server
  • Manuals and Tutorials
  • Examples

Contribution guidelines

Before you make your first contribution it is recommended that you familiarize yourself with our contribution guidelines. The guidelines exist to ensure that your contribution is of good quality, does what it is supposed to do, has no unwanted side-effects and that you are legally allowed to make the contribution. Below is a short summary of the guidelines. Head over to GitHub to read the full guidelines.

Bugfixes

  • Does the contribution solve a single problem?
  • Is the contribution covered by unit tests?
  • Is the contribution tested on all applicable platforms?
  • Is the contribution following existing code style and patterns?
  • Are you legally entitled to make the contribution?

New features

  • Does the contribution add a single feature?
  • Is the new functionality well designed?
  • Is the contribution covered by unit tests?
  • Is the contribution tested on all applicable platforms?
  • Is the contribution following existing code style and patterns?
  • Are you legally entitled to make the contribution?
